# 구현 기능별 검증 가능성 분석

## 한 줄 결론

미리(MIRI)의 MVP는 “그럴듯한 입시 조언”이 아니라 **근거가 붙은 진단·로드맵·면접 질문을 얼마나 안정적으로 만들 수 있는지**를 검증해야 한다. 기능 우선순위는 `기술 구현 가능성`보다 `근거 검증 가능성`, `법무 리스크`, `사용자 결제 검증 가능성`을 함께 봐야 한다.

## 판정 기준

| 기준 | 의미 |
|---|---|
| 구현 난이도 | 2~6주 안에 데모 가능한지 |
| 검증 가능성 | 정답/전문가 평가/사용자 행동으로 품질을 검증할 수 있는지 |
| 법무 리스크 | 개인정보, 미성년자, 자동화된 결정, 합격 보장 오해 가능성 |
| MVP 판정 | 지금 넣기, 제한 파일럿, V2, 제외 |

## 지금 넣어도 되는 기능

### 1. 정규표현식 기반 JSON 섹션 분리

- 검증 방법: 원문 20~50개를 사람이 라벨링하고, AI/정규식 결과와 비교한다.
- 성공 기준: 주요 섹션 분리 정확도 90% 이상, 누락/오분류 항목 사람이 확인 가능.
- 실패 시 대응: 완전 자동 대신 “분리 실패 문장 확인 화면”을 둔다.

### 2. 학생부 진단

- 검증 방법: 입시 컨설턴트 2명에게 동일 학생부 진단 결과를 블라인드 평가시킨다.
- 성공 기준: 근거 없는 주장 0건, 전문가 유용성 평균 4점 이상, 핵심 약점 누락률 20% 이하.
- 주의 문구: “공식 평가가 아닌 학생부종합전형 대비 참고 진단입니다.”

### 3. 전공/학과 적합도

- 검증 방법: 희망학과별 전공 키워드와 학생부 근거가 실제로 연결됐는지 평가한다.
- 성공 기준: 적합 근거 3개 이상, 부족 근거 2개 이상 제시, 사용자가 납득한 비율 70% 이상.
- 실패 시 대응: 점수보다 “맞는 근거/부족한 근거” 중심으로 보여준다.

### 4. 성장 로드맵

- 검증 방법: 추천 활동이 실제 학기 안에 가능한지 학생/학부모/컨설턴트에게 물어본다.
- 성공 기준: 실행 가능성 평균 4점 이상, 과장 활동 추천 0건, 원문 근거 연결률 90% 이상.
- 주의 문구: “활동 보장”이 아니라 “활동 아이디어와 준비 방향”으로 표현한다.

### 5. 면접 예상 질문

- 검증 방법: 질문이 학생부 근거에서 나왔는지, 꼬리질문이 자연스러운지 평가한다.
- 성공 기준: 근거 기반 질문 비율 90% 이상, 전문가가 “실제 면접 대비에 쓸 만함” 4점 이상.
- MVP 가치: 가장 빠르게 품질 검증이 가능하고 법무 리스크가 낮다.

## 제한 파일럿으로만 넣을 기능

### 1. 빈틈 전형 후보 추천

- 왜 제한해야 하나: 대학별 시행계획과 최종 모집요강은 바뀔 수 있고, PDF 구조도 대학마다 다르다.
- 검증 방법: 30~50개 대학, 20개 전형군만 먼저 구조화한다.
- 성공 기준: 자격조건 추출 정확도 95% 이상, 출처 URL/업데이트 날짜 100% 표시.
- 주의 문구: “지원 가능성이 있는 후보”이지 “지원 가능 확정”이 아니다.

### 2. 미래 생기부 미리보기

- 왜 제한해야 하나: 공식 NEIS 기록처럼 오인되면 리스크가 크다.
- 검증 방법: 문장별 원문 근거 연결, 금지/과장 표현 검사, 교사/컨설턴트 리뷰.
- 성공 기준: 근거 없는 문장 0건, 공식 기록 오해 문구 0건.
- 제품 위치: 메인 기능이 아니라 로드맵 결과를 보여주는 샘플.

## V2로 미룰 기능

### 1. 특수 진로/예체능 로드맵

- 이유: 악기, 종목, 실기, 포트폴리오, 전문가 네트워크가 필요하다.
- 검증 방법: 한예종/서울예대/KUSF 축으로만 작은 DB를 만들고 전문가 1명과 검수한다.
- V2 조건: 전문가 검수 또는 파트너 상담 모델이 붙을 때.

### 2. 특목고 정보/고입 코치

- 이유: 중학생 학부모 타겟이라 대학 학생부종합전형 코치와 고객군이 다르다.
- 검증 방법: 별도 랜딩페이지로 수요를 확인한다.
- V2 조건: 고입 자기소개서/면접 코칭 수요가 별도로 확인될 때.

## 제외하거나 표현을 바꿀 기능

### 합격 가능성 시뮬레이션

- 문제: 정확한 합격/불합격 예측은 데이터가 부족하고 법무·신뢰 리스크가 크다.
- 대체 표현: “전형 리스크 분석”, “지원 전략 참고”, “추가 확인 필요”.
- 출시 조건: 충분한 합격/불합격 데이터, 대학별 전형 변화 추적, 전문가 검수 체계가 있을 때.

## 반드시 지켜야 할 법무/운영 가드레일

### 개인정보

- 만 14세 미만 아동의 개인정보를 처리하기 위해 동의가 필요한 경우 법정대리인 동의와 확인이 필요하다.
- 학생부에는 이름, 학교, 학년, 활동, 진로, 경우에 따라 사회통합/특수교육 관련 정보가 들어갈 수 있으므로 최소 수집이 원칙이다.
- 주민등록번호, 건강, 장애, 가족관계, 소득 관련 증빙은 MVP에서 직접 받지 않는 편이 안전하다.
- 업로드 파일은 기본적으로 “분석 후 즉시 삭제” 옵션을 둔다.

### AI/자동화된 결정

- AI가 학생의 권리·의무에 중대한 영향을 주는 자동화된 결정을 내리는 것처럼 보이면 안 된다.
- 모든 결과는 “참고”, “후보”, “검토 필요”로 표현하고, 사람이 최종 판단해야 한다.
- 진단/추천마다 근거와 설명을 제공해야 한다.

### 생성형 AI 고지

- 결과물이 AI로 생성되었다는 사실을 명확히 고지한다.
- 생성 결과를 공식 학교생활기록부, 공식 입시 판단, 합격 보장으로 오인하게 만들지 않는다.

8. 미래 생기부 미리보기
9. 특수 진로/특목고 vertical
10. 합격 가능성 시뮬레이션

## 멘토에게 검증받을 질문

1. 첫 MVP에서 `빈틈 전형 후보 추천`을 넣는 것이 결제 전환에 도움이 되는가, 아니면 데이터 부담만 커지는가?
2. `학생부 진단 + 로드맵 + 면접 질문`만으로 3만 원대 유료 리포트가 가능한가?
3. 미성년자 학생부 데이터를 저장하지 않고 “분석 후 삭제”로 가면 초기 검증에 충분한가?
4. AI기본법상 교육 영역 고영향 AI 오해를 피하려면 어떤 표현과 고지 문구가 필요한가?
