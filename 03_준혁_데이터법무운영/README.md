## 본인 미션 (이 4가지만 답하면 됨)

### 1. 데이터 소유 플랫폼 사례 (60분)

1차 멘토가 *"정보가 개인에게 귀속되는 형태로 가야 한다"* 라고 짚은 컨셉. 이걸 구현한 사례 찾기:
- **국내 마이데이터 사례**: 헬스케어(국민건강보험), 금융(마이데이터 사업자)
- **해외 사례**: Notion(개인 데이터), Solid(팀 버너스리), 1Password 등
- **교육 분야**: 학원이 학생 데이터를 가져가는 현 구조 vs 학생 소유 모델
- **기술적 구현 방식**: 어떻게 학생이 자기 데이터를 진짜 소유하게 할까

→ 산출물: `01_데이터소유_플랫폼_사례.md`

### 2. 미성년자 개인정보 법적 검토 (60분)

학생 데이터를 모을 때 반드시 알아야 할 법:
- **개인정보보호법** 미성년자 관련 조항
- **학원의 학습자 개인정보 처리** 가이드라인 (교육부)
- **만 14세 미만 / 14세 이상** 처리 차이
- **부모 동의** 필수 영역
- **데이터 보관·파기** 의무
- 우리 서비스가 가장 주의해야 할 5가지 리스크

→ 산출물: `02_개인정보_법무_검토.md`

> ⚠ 법적 부분은 절대 추측하지 말 것. 출처 불명확하면 ❓ 표시 후 변호사 검토 필요로 마무리.

### 3. 부모 광고 채널 (40분)

1차 멘토가 *"부모를 타겟으로 광고해야"* 라고 짚음. 어디가 효과적인가:
- 맘카페 (네이버 카페 / 다음 카페)
- 인스타그램 광고 (학부모 타겟팅)
- 카카오톡 채널
- 유튜브 (입시 콘텐츠)
- 학원 제휴
- 각 채널의 CPC·전환률 사례

→ 산출물: `03_부모_광고_채널.md`

### 4. 무료→유료 전환 모델 (40분)

1차 멘토가 *"무료로 풀고 유저 모은 다음 수익화"* 권유. 사례 조사:
- 한국 입시·교육 분야 freemium 사례
- 무료에서 어디까지 주고 어디부터 유료인가의 패턴
- 유료 전환률 (한국 시장)
- 사전예약 / 알림 신청 / 베타 모델 비교

→ 산출물: `04_무료유료_전환_모델.md`

### 5. 결론 1페이지 (30분, 마지막에)

위 4개를 한 페이지로 압축. `templates/결론_1페이지_템플릿.md` 사용.

→ 산출물: `00_결론_1페이지.md`
