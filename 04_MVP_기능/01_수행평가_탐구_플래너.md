# MVP 1순위: 수행평가 탐구 플래너

## 결론

`수행평가 탐구 플래너`는 MIRI의 첫 진입 기능으로 가장 적합하다. 학생이 받은 막연한 수행평가를 **자료 조사, 교과 개념 연결, 탐구 질문, 개요 작성, 발표 준비, 제출 전 확인**으로 쪼개준다. 완성 문장을 대신 쓰지 않기 때문에 대필 리스크가 낮고, 학생과 학부모가 바로 효용을 느끼기 쉽다.

## 사용자 문제

- 학생은 수행평가 주제를 받아도 어디서 시작해야 할지 모른다.
- 학부모는 아이가 실제로 무엇을 하고 있는지 확인하기 어렵다.
- 교사는 학생이 수업 안에서 스스로 질문하고 탐구하는 과정을 더 중요하게 본다.

## 입력값

| 입력 | 예시 |
|---|---|
| 과목 | 통합사회, 영어, 확률과 통계, 생명과학 |
| 수행평가 설명 | `AI 윤리 관련 발표`, `의료 통계 분석 보고서` |
| 현재 단원 | `인공지능과 사회`, `조건부확률`, `생명공학` |
| 희망 진로 | 컴퓨터공학, 의료 AI, 경영, 심리 |
| 학생 수준 | 자료 조사부터 필요 / 개요는 있음 / 발표만 남음 |
| 마감일 | 2026-05-12 |

## 출력값

| 출력 | 설명 |
|---|---|
| 탐구 목표 | 이번 수행평가에서 보여줄 역량을 1줄로 정리 |
| 단계별 TODO | 15~30분 단위의 행동 체크리스트 |
| 교과 개념 연결 | 수행평가가 어떤 수업 개념과 연결되는지 표시 |
| 탐구 질문 후보 | 학생이 직접 고를 수 있는 질문 3~5개 |
| 개요표 | 서론/본론/결론이 아니라 조사 흐름을 표로 제시 |
| 추천 근거 | 왜 이 TODO가 필요한지 설명 |

## 예시 흐름

1. 학생이 수행평가 안내문을 붙여넣는다.
2. AI가 과목, 마감일, 평가 기준, 산출물 형식을 추출한다.
3. AI가 `오늘 할 일 1개`와 `이번 주 할 일 3개`로 쪼갠다.
4. 학생이 관심 키워드 1~2개를 직접 선택한다.
5. AI가 탐구 질문 후보와 개요표를 제시한다.
6. 학생이 완료 체크를 하면 다음 심화 TODO로 넘어간다.

## 절대 하지 말 것

- 보고서 문단을 대신 작성하지 않는다.
- 세특에 들어갈 문장을 생성하지 않는다.
- `합격에 유리한 표현`처럼 보이는 미사여구를 붙이지 않는다.
- 교사가 관찰하지 않은 활동을 만들어내지 않는다.

## 검증 지표

| 지표 | 목표 |
|---|---|
| 첫 TODO 생성 완료율 | 70% 이상 |
| 첫 TODO 체크 완료율 | 40% 이상 |
| 학생 만족도 | “시작하기 쉬워졌다” 응답 70% 이상 |
| 학부모 결제 의향 | 3만 원대 1회 플래너 결제 의향 확인 |
| 대필 오해율 | 인터뷰에서 대필로 오해한 응답 최소화 |

## MVP 화면

- 수행평가 입력 화면
- 추출된 평가 조건 카드
- 오늘 할 일 1개
- 이번 주 TODO 3개
- 탐구 질문 후보
- 추천 근거 보기
- 완료 체크 및 다음 TODO
