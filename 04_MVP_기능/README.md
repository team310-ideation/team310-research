# MIRI MVP 기능 폴더

## 한 줄 결론

MIRI의 MVP는 `생기부 문장 생성`이 아니라 **수행평가와 교과 탐구를 학생이 직접 설계하도록 돕는 AI 탐구 플래너**다. AI는 완성 문단을 쓰지 않고, 질문·개요·체크리스트·다음 행동·추천 근거를 제공한다.

## MVP 필수 기능 3개

| 우선순위 | 기능 | 역할 | 규제 리스크 | MVP 판단 |
|---:|---|---|---|---|
| 1 | 수행평가 탐구 플래너 | 막막한 수행평가를 단계별 TODO로 분해 | 낮음 | 지금 개발 |
| 2 | 탐구 질문 만들기 | 수업 중 활용 가능한 심화 질문 생성 | 낮음 | 지금 개발 |
| 3 | 지식 확장형 꼬리 질문 로드맵 | 완료 활동에서 다음 탐구로 연결 | 낮음 | 지금 개발 |

## 제한 기능

| 기능 | 판단 | 이유 |
|---|---|---|
| 학종 부합도 진단 | 제한 개발 | 학생이 쓴 초안의 누락/비약만 경고. 문장 재작성 금지 |
| 월말 성장 리캡 | 리텐션 기능 | 학부모 결제 유지에는 중요하지만 MVP 핵심 학습 가치는 1~3순위보다 낮음 |
| 미래 생기부 미리보기 | 제한 파일럿 | 공식 학생부 문장처럼 보이면 위험. 역량 근거 시각화로 표현 |

## 공통 UX 원칙

- 완성 문단 생성 금지: 불릿, 개요, 질문, 체크리스트로만 출력
- 학생 입력 필수: 학생이 자기 생각/키워드/자료를 넣어야 다음 단계 활성화
- 추천 근거 표시: 모든 TODO에 `왜 이 행동을 추천했는가` 표시
- 교과 본질 우선: 모든 과목을 억지로 진로에 연결하지 않기
- 결과 보장 금지: 합격 가능성, 세특 완성, 명문대 보장 표현 금지
- 무료 베타 우선: 초기에는 결제보다 리포트 품질과 사용자 반응 검증
- 아카이빙 가치: 단발 답변이 아니라 학생 기록 누적과 전략 갱신을 핵심 차별점으로 둠
