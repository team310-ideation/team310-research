# 휴포지토리 비전

## 한 줄 정의

휴포지토리는 개인 기록을 잘 정리해 두고, AI가 그 기록을 바탕으로 현재의 나를 이해하고 다음 선택을 돕는 **Life Designer** 비전이다.

## 문제의식

- 행복의 핵심은 인간관계라는 관점에서 출발한다.
- 세종의 장기 비전은 100년 인생을 AI와 함께 설계하는 라이프 디자인 서비스다.
- 풀고 싶은 문제는 크게 두 가지다.
  - 인간관계: 누구를 만날 것인가
  - 삶의 목적: 무엇에 몰입할 것인가

## 작동 원리

1. 일기, 블로그, 프로젝트 기록을 마크다운으로 누적한다.
2. GitHub Repository에 업로드해 개인 기록 저장소를 만든다.
3. AI가 기록을 바탕으로 미래 선택을 예측하거나 제안한다.
4. 현실에서 실험한다.
5. 피드백을 다시 AI에 입력해 맥락을 업데이트한다.

## 개인 실험 사례

- 소마 합격 예측: 본인 블로그 데이터를 AI에 넣고 합격 전략을 질문한 뒤, 유튜브 운영 추천을 받아 실행했다.
- 팀 빌딩: 타 연수생 데이터를 AI에 입력해 팀 핏을 분석했다.

## 개인 운영 방식

- VS Code + Markdown + YAML을 결합한다.
- AI가 폴더 구조를 자동 설계한다.
- Daily Layer와 Private Layer를 분리한다.
  - Daily Layer: 일기, 점만 찍는 가벼운 기록
  - Private Layer: 프로젝트별 깊은 문서
- 일기 입력 시 AI가 은평구 청년축제, 소마온, 팀310 등 적절한 폴더로 자동 분류한다.

## 일기 시장의 한계

- 시장 검증 결과, 대부분의 사람은 일기를 쓰지 않는다.
- 멘토와 VC들은 개인 기록 서비스 단독 시장성에 부정적이었다.
- 맘스 다이어리처럼 강한 동기와 보상이 있는 구조는 참고할 만하다.
  - 100일 일기 작성 시 무료 책 출판
  - 중도 포기자용 부활 쿠폰 판매

## 피보팅 히스토리

1. 개인 기록 휴포지토리: 시장성 부족.
2. 멘토링 대화 기록 자동 저장: 일기를 쓰지 않아도 대화가 자동 누적되는 MVP 형태.
3. B2B 우수 코치 양성 시스템: 멘토·멘티 양방향 데이터 누적.
4. 웹소설 기반 콘텐츠화: 혜정 합류 후 검토했으나 이탈로 변동.

## 미리와의 관계

- 미리는 휴포지토리의 첫 적용 사업이다.
- 휴포지토리의 "개인 기록 + AI 맥락 업데이트" 구조를 청소년 생기부와 진로 선택 문제에 적용한다.
