# AGENTS.md — AI 에이전트 사용 가이드 (3명 호환용)

**목적**: 3명이 서로 다른 AI를 써도 결과물이 같은 품질·같은 양식으로 나오도록.

---

## 0. 공통 원칙

1. **항상 한국어로 답하게 하기** — 영어 답변 받으면 즉시 "한국어로 다시" 요청
2. **출처 명시 강제** — 답변 끝에 *"위 내용의 출처를 링크 또는 검색 키워드로 명시해줘"* 추가
3. **모르면 모른다고 답하게** — *"확실하지 않은 부분은 ❓ 표시 후 답해"* 프롬프트에 포함
4. **마크다운 출력 강제** — *"마크다운 형식으로, 헤딩·표·리스트 활용해서 답해"*

## 1. 트랙별 시작 프롬프트 (복붙해서 바로 쓰기)

### 🅰 세민 (시장·경쟁사 트랙) — GPT/Codex용

```
나는 한국에서 중·고등학생 진로/생기부 관리 AI 서비스 '미리(MIRI)'를 만드는 팀이야.
오늘 4시간 안에 2차 멘토 미팅이 있고, 너는 내 시장·경쟁사 조사를 맡고 있어.

조사할 것:
1. 한국 입시 컨설팅·생기부 관리 분야 주요 경쟁사 5~7개의 기능·가격·약점
   (진학사, 메가스터디, 김캐디, 코멘토, 클라썸, 튜링, 에듀베이터 등)
2. 우리 서비스가 일반 ChatGPT 대비 가질 수 있는 진짜 차별점 (학생 데이터 누적 활용)
3. '미래 생기부 미리보기' 컨셉이 시장에서 받는 평가 가능성

각 항목에 대해 다음 양식으로 답해줘:
- ## 결론 (2~3줄)
- ### 근거 (출처 링크 또는 검색 키워드 첨부)
- ### ❓ 멘토에게 물어볼 것 (확실하지 않은 부분)

마크다운으로 답하고, 모르는 건 만들어내지 말고 ❓ 표시.
```

### 🅱 세종 (빈틈 전형·특수 진로 트랙) — VS Code Codex용

```
나는 한국에서 중·고등학생 입시 AI 서비스 '미리(MIRI)'를 만드는 팀이야.
2차 멘토 미팅이 4시간 후에 있고, 너는 내 '빈틈 전형'과 '특수 진로 트랙' 조사를 맡고 있어.

조사할 것:
1. 한국 대학 입시에서 잘 알려지지 않은 '빈틈 전형' 사례
   - 외국인 정원외, 재외국민, 농어촌학생, 특수재능 등
   - 한두 명만 뽑는 특수전형 사례
   - 어떤 대학이 어떤 전형을 운영하는지 데이터 수집 방법
2. 한예종/예술 특기/체육 특기 등 비전형 입시의 시장 규모와 진입 방식
3. 특목고(외고/과학고/영재고/자사고) 진학 정보 시장의 정보 부족 현황

각 항목에 대해:
- ## 결론
- ### 데이터 출처 (어디서 어떻게 모을 수 있는지)
- ### ❓ 멘토에게 검증받을 것

마크다운으로, 출처 없으면 신뢰도 낮음으로 표기.
```

### 🅲 준혁 (데이터·법무·운영 트랙) — Gemini/Claude용

```
나는 한국에서 중·고등학생 입시 AI 서비스 '미리(MIRI)' 팀이야.
2차 멘토 미팅이 4시간 후이고, 너는 데이터·법무·운영 영역 조사 담당이야.

조사할 것:
1. '학생 개인이 자기 입시 데이터를 소유하는 플랫폼' 사례 (국내외)
   - 헬스케어의 마이데이터, 금융의 마이데이터처럼
2. 미성년자 개인정보 처리 법적 검토 (개인정보보호법, 학생 데이터)
   - 우리 서비스가 학생 생기부·활동 데이터 모을 때 주의할 점
3. 한국 학부모 대상 광고 효과적인 채널
   - 맘카페, 인스타그램, 유튜브, 카카오톡 채널 등 비교
4. 입시 서비스의 무료→유료 전환 모델 사례
   - 어느 시점에 어떤 가치로 유료화하는 게 효과적인가

각 항목에 대해:
- ## 결론
- ### 근거 / 사례
- ### ❓ 멘토에게 물어볼 것 (특히 법무는 변호사 확인 필요)

마크다운으로, 법적 부분은 절대 추측하지 말고 ❓ 표시.
```

## 2. 도중에 막힐 때 쓸 프롬프트

### 출처가 부족할 때
```
방금 답변에서 출처가 명확하지 않은 부분이 있어. 다음 항목에 대해 다시 조사해서
출처 링크 또는 정확한 검색 키워드를 알려줘. 모르면 ❓ 표시.

[항목]
```

### 결론이 추상적일 때
```
방금 답변이 "보편적이고 추상적"이라는 피드백을 받았어. 더 구체적인 사례,
숫자, 회사 이름, 가격 등을 넣어서 다시 답해줘. 막연한 표현 ("관련 분야",
"전반적으로" 등) 금지.
```

### 결론 1페이지로 압축
```
지금까지 조사한 내용을 한 페이지로 압축해줘. 양식:

# [트랙명] 결론 1페이지

## 핵심 발견 3가지
1. ...
2. ...
3. ...

## 우리가 가져갈 수 있는 것
- ...

## 우리가 가져갈 수 없는 것 (이유)
- ...

## ❓ 멘토에게 물어볼 것 (3개 이내)
- ...
```

## 3. 결과물 검증 (Output Taste)

본인이 받은 AI 답변을 그대로 커밋하지 말 것. 다음 체크리스트:

- [ ] 추상적 표현 (보편적, 일반적, 전반적) 빼고 구체화했나?
- [ ] 출처 링크 또는 검색 키워드 있나?
- [ ] 한국 시장 사례 위주인가? (해외 사례만 있으면 약함)
- [ ] ❓ 표시된 항목이 3개 이내인가? (너무 많으면 추가 조사)
- [ ] 결론이 2~3줄로 요약되나?

→ 통과 못 하면 **AI에게 다시 시키기.** 본인이 직접 다듬느라 시간 쓰지 말 것.

## 4. 서로 다른 AI 답변이 충돌할 때

3명이 같은 주제를 서로 다른 AI에게 물어볼 가능성도 있어. 충돌 시:

1. 둘 다 출처 링크 비교
2. 더 최신 데이터 가진 쪽 우선
3. 출처 둘 다 약하면 → ❓ 멘토에게 물어볼 것에 추가
