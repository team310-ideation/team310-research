# AI 차별점: ChatGPT가 아니라 학생부종합전형 코치

## 한 줄 결론

일반 ChatGPT는 “그럴듯한 조언”을 만들 수 있지만, 미리(MIRI)는 **학생의 실제 기록을 정규식으로 구조화하고, Student Graph에 누적해 종합전형 진단·전공 적합도·활동 로드맵·면접 대비를 연결**한다. 차별점은 모델 자체보다 **학생 데이터 구조화, 근거 연결, 다음 행동 추천**이다.

## 핵심 발견 3가지

### 1. 차별점은 LLM이 아니라 Student Graph다

- 근거: 일반 ChatGPT는 사용자가 매번 맥락을 설명해야 한다. 미리는 학생별로 `과목`, `활동`, `진로`, `교사 문체`, `반복 키워드`, `강점/약점`을 축적할 수 있다.
- 출처: 내부 서비스 설계 가설. 비교 출처는 교육부 2026 학교생활기록부 기재요령 공지: https://www.moe.go.kr/boardCnts/viewRenew.do?boardID=316&boardSeq=105372&lev=0&m=030215&opType=N&s=moe&statusYN=W

### 2. “생성”보다 “근거 있는 코칭”이 제품 가치다

- 근거: 생기부 문장은 교사 관찰·수업·평가 기록과 연결되어야 한다. 따라서 미리는 문장마다 `basis`, `basis_type`, `source_section`을 붙여야 한다.
- 출처: 교육부 2026학년도 학교생활기록부 기재요령 안내, 학교생활기록부 종합지원포털 Q&A: https://star.moe.go.kr/web/contents/m40100.do?id=107757&schM=view

### 3. 경쟁 컨설팅 업체도 “AI 단독 생성의 위험”을 말한다

- 근거: 메드스카이는 AI가 탐구 주제를 그럴듯하게 만들 수 있어도 학생의 이해도, 정확성, 생기부 흐름과의 적합성 검토가 필요하다고 설명한다.
- 출처: https://www.medsky.co.kr/management

## 일반 ChatGPT vs 미리(MIRI)

| 항목 | 일반 ChatGPT | 미리(MIRI) |
|---|---|---|
| 입력 | 사용자가 자유롭게 붙여넣은 텍스트 | OCR/JSON을 정규표현식으로 섹션 분리 |
| 기억 | 세션마다 불안정 | 학생별 누적 Student Graph |
| 출력 | 일반론/문장 예시 | 진단, 적합도, 로드맵, 면접 질문, 생기부 초안 |
| 근거 | 사용자가 따로 확인 | 문장마다 근거 섹션 연결 |
| 검증 | 사용자가 직접 | 금지 표현, 과장 표현, 근거 없는 단정, 합격 보장 표현 검사 |
| 사용 가치 | 글쓰기 도우미 | 종합전형 대비 상태를 보고 다음 행동을 정하는 코치 |

## 우리가 가져갈 수 있는 것

- `학생 기록 → 구조화 JSON → Student Graph → 진단/로드맵/면접/초안` 파이프라인을 제품 핵심으로 둔다.
- AI-only 리포트의 약점은 실행 관리와 동기 부여다. 따라서 미리는 장기적으로 `AI 진단 + 전공 멘토 실행 관리`를 붙여 일반 ChatGPT와 AI 리포트형 경쟁사 모두와 차별화한다.
- 출력 JSON에는 반드시 다음 필드를 넣는다.

```json
{
  "module": "activity_roadmap",
  "target_major": "컴퓨터공학과",
  "recommendation": "확률과 통계 수업의 데이터 분석 수행평가를 진로 탐구로 확장한다.",
  "basis": ["2학년 수학 세특", "진로활동 기록"],
  "expected_output": ["탐구 보고서", "면접 소재", "세특 반영 가능 소재"],
  "basis_type": "source_based_inference",
  "needs_human_review": true
}
```

- “AI가 생기부를 써준다”가 아니라 **“학생 기록 기반으로 종합전형 대비 방향을 코칭한다”**로 표현한다.
- 미래 생기부 문장 생성은 로드맵의 결과를 미리 보여주는 보조 기능으로 둔다.
- 대학생 멘토는 전체 매칭 플랫폼으로 바로 확장하지 않고, 초기에는 검증 멘토 5~10명이 AI 리포트를 보고 30분 피드백하는 수동 실험으로 검증한다.

## 우리가 가져갈 수 없는 것

- ❌ 공식 NEIS 기록을 자동 생성/대체한다고 말하지 않는다.
- ❌ 교사 관찰 없이 확정 사실처럼 기재하지 않는다.
- ❌ 출처 없는 대회, 수상, 외부활동, 기관명, 실적을 만들어내지 않는다.
- ❌ 대학 합격 가능성을 확정적으로 말하지 않는다.
- ❌ 대학생 멘토가 전문 컨설턴트처럼 최종 합격 전략을 보장한다고 말하지 않는다. 멘토는 실행 관리와 경험 기반 피드백 역할로 제한한다.

## ❓ 멘토에게 검증받을 것

- ❓ “AI 학생부종합전형 코치”가 학교생활기록부 기재요령과 충돌하지 않게 표현되는 문구는 무엇인가?
- ❓ B2C 학부모용으로 먼저 갈지, B2B 컨설턴트/학원용 보조도구로 먼저 갈지?
- ❓ 학생 기록을 장기 저장할 때 동의/보관/삭제 정책을 어디까지 준비해야 하는지?

## AI 사용 메모

- 사용 프롬프트:
  ```text
  일반 ChatGPT와 학생별 기록 기반 학생부종합전형 코치의 차별점을 제품 관점에서 정리해줘. 공식 생기부 대체가 아닌 진단/로드맵/면접 대비 서비스로 표현해줘.
  ```
- 답변에서 신뢰도 낮았던 부분: 실제 학교 현장에서 허용되는 AI 보조 범위
