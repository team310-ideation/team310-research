# AI 학생부종합전형 코치 MVP 기획안

## 서비스 한 줄 정의

미리(MIRI)는 학생부와 활동 기록을 분석해 **종합전형 대비 상태를 진단하고, 희망 전공에 맞는 보완 활동과 면접 준비를 추천하는 AI 코치**다.

## 첫 타겟

- 1순위: 고1~고2 학생·학부모
- 이유: 아직 학생부를 보완할 시간이 있어 로드맵 기능의 가치가 크다.
- 2순위: 고3 학생·학부모
- 이유: 면접 질문, 자기 기록 정리, 원서 전 점검 수요가 있다.
- 3순위: 입시 컨설턴트/학원
- 이유: 여러 학생을 빠르게 진단하는 B2B 보조도구로 확장 가능하다.

## MVP 핵심 기능 4개

### 1. 학생부 진단

- 입력: 생기부 JSON, OCR 텍스트, 활동 기록
- 출력: 강점, 약점, 부족한 평가 요소
- 기준: 학업역량, 진로역량, 공동체역량, 성장성
- 차별점: 문장마다 원문 근거를 연결한다.

### 2. 전공/학과 적합도

- 입력: 희망학과, 희망진로, 학생부 기록
- 출력: 적합도 요약, 맞는 근거, 부족한 근거
- 예시: “컴퓨터공학 희망인데 수학 탐구 근거는 있으나 협업 프로젝트 근거가 약함.”

### 3. 생기부 성장 로드맵

- 입력: 진단 결과와 희망 전공
- 출력: 이번 학기 과목별 세특 소재, 탐구 주제, 활동 계획
- 원칙: 이미 한 활동을 부풀리지 않고, 앞으로 할 수 있는 행동을 제안한다.

### 4. 면접 예상 질문

- 입력: 학생부 진단 결과와 핵심 활동
- 출력: 예상 질문, 꼬리질문, 답변 방향, 약점 질문
- 차별점: 학생 기록 근거를 눌러 확인할 수 있게 만든다.

## 보조 기능: 미래 생기부 미리보기

미래 생기부 미리보기는 메인 기능이 아니라 로드맵의 결과를 보여주는 보조 기능으로 둔다.

- 좋은 표현: “추천 활동이 기록화되면 어떤 식으로 보일 수 있는지 참고 초안을 제공합니다.”
- 피해야 할 표현: “AI가 NEIS 생기부를 대신 작성합니다.”

## 화면 흐름

1. 학생 정보 입력: 학년, 희망학과, 희망진로
2. 자료 업로드: 생기부 JSON/OCR 텍스트/활동 기록
3. 정규표현식 파싱: 과목, 창체, 진로, 행특, 활동 기록 분리
4. 진단 결과: 강점·약점·근거 카드 표시
5. 전공 적합도: 희망학과와 현재 기록 매칭
6. 로드맵: 이번 학기 할 일 추천
7. 면접 대비: 학생부 기반 질문 생성
8. 결과 출력: Markdown/PDF/JSON 리포트

## 데이터 구조 초안

```json
{
  "student_profile": {
    "grade": 2,
    "target_major": "컴퓨터공학과",
    "target_career": "AI 개발자"
  },
  "diagnosis": {
    "strengths": ["수학 탐구", "문제 해결"],
    "weaknesses": ["협업 프로젝트 근거 부족"],
    "evidence": ["2학년 수학 세특", "진로활동 기록"]
  },
  "roadmap": [
    {
      "action": "확률과 통계 수행평가를 데이터 분석 탐구로 확장",
      "expected_record_area": "수학 세특",
      "basis": ["2학년 수학 세특"],
      "needs_human_review": true
    }
  ],
  "interview_questions": [
    {
      "question": "데이터 분석 탐구에서 가장 어려웠던 점은 무엇이었나요?",

    }

```

## 유료화 가설

| 상품 | 가격 가설 | 제공 내용 |
|---|---:|---|
| 무료 진단 Lite | 0원 | 강점 3개, 약점 3개, 분석 가능 항목 수 |
| 전략 리포트 | 29,000~39,000원 | 진단, 전공 적합도, 로드맵, 면접 질문 |
| 코치 패키지 | 99,000~199,000원 | 학기별 계획, 답변 피드백, 컨설턴트 검수 옵션 |

## 2주 안에 만들 수 있는 데모

1. 샘플 생기부 JSON 업로드
2. 정규표현식 기반 섹션 분리
3. 학생부 진단 카드 생성
4. 희망학과 입력 후 적합도 생성
5. 로드맵 5개와 면접 질문 10개 생성
6. Markdown 리포트 다운로드

## 리스크

- 공식 생기부 생성처럼 보이면 법무/윤리 리스크가 커진다.
- 합격 가능성을 단정하면 신뢰와 규제 리스크가 생긴다.
- 학생 개인정보 보관·삭제·동의 정책이 없으면 B2B 확장이 어렵다.
