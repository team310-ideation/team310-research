# MIRI 최종 보고서: AI 수행평가·탐구 플래너 전략

## 1. 결론

MIRI는 `생기부 생성기`나 `세특 대필기`가 아니라 **수행평가와 교과 탐구를 학생이 직접 설계하도록 돕는 AI 학종 프로젝트 매니저**로 포지셔닝해야 한다. AI의 출력은 완성 문단이 아니라 체크리스트, 질문, 개요, 다음 행동으로 제한해야 한다.

2026학년도 학생부 기재요령과 학종 평가 기조를 고려하면, AI가 학생부 문장이나 보고서를 대신 작성하는 방향은 위험하다. 반대로 학생이 수업 시간 안에서 탐구 동기, 질문, 자료 조사, 지식 확장, 성취 과정을 스스로 보여주도록 돕는 `인지적 코치`는 규제 리스크가 낮고 학종 평가 방식에도 잘 맞는다.

따라서 6개월 MVP는 `수행평가 탐구 플래너 + 탐구 질문 만들기 + 지식 확장형 꼬리 질문 로드맵` 3개에 집중하고, `제출 전 AI 진단`, `월말 성장 리캡`, `미래 생기부 미리보기`는 제한 기능 또는 리텐션 기능으로 뒤에 붙이는 전략이 가장 현실적이다.

|---|---|---|---|---|
| 대형 입시 브랜드 | 진학사, 메가스터디, 대형 컨설팅 센터 | 원서, 합격예측, 상담, 브랜드 신뢰 | 평소 학생부 활동을 매주 관리하는 UX는 약함 | 학생부 기반 진단과 행동 로드맵 |

| 특수 진로/vertical | 메디컬 컨설팅, 예체능 입시 학원, 체대 입시 학원 | 전문성, 실기/면접/서류 노하우 | 정보가 흩어져 있고 전문가 의존도가 높음 | 서류·면접·환산표·일정 관리 보조 |

## 3. MIRI의 핵심 차별점

### 3.1 일반 ChatGPT 대비 차별점

| 항목 | 일반 ChatGPT | MIRI |
|---|---|---|
| 입력 데이터 | 사용자가 매번 맥락 설명 | 학생부, 활동 기록, 희망 전공, 전형 조건을 누적 |
| 결과물 | 범용 조언과 문장 생성 | 질문, 개요, 체크리스트, 추천 근거가 붙은 다음 행동 |
| 실행 관리 | 대화 이후 추적 어려움 | 수행평가 TODO, 탐구 질문, 심화 로드맵, 진행률 추적 |
| 학부모 가치 | 결과를 신뢰하기 어려움 | 학부모 리포트, 위험 신호, 멘토 검수 큐 |
| 법무 안전성 | 대필·합격 보장 오해 가능 | 준비 상태, 누락 항목, 확인 필요 사항 중심 |

MIRI의 차별화 언어는 `AI가 생기부를 써준다`가 아니라 **AI가 수행평가와 탐구 과정을 쪼개 학생이 직접 행동하게 만든다**여야 한다.

### 3.2 제품 언어

- 핵심 포지션: `AI 수행평가·탐구 플래너`
- 더 안전한 설명: `근거 기반 학종 준비 프로젝트 매니저`
- 학부모용 한 줄: `우리 아이가 수행평가와 교과 탐구를 막막해하지 않도록 단계별 계획을 잡아주는 서비스`
- 학생용 한 줄: `수행평가를 어디서부터 시작할지 질문, 자료 조사, 개요, 다음 행동으로 쪼개주는 코치`
- 멘토용 한 줄: `AI가 만든 진단과 태스크를 검수하고 실행을 붙잡는 관리 도구`

## 4. 6개월 MVP 제안

### 결론

6개월 MVP는 거대한 입시 플랫폼이 아니라 **수행평가와 교과 탐구를 단계별 행동으로 바꾸는 얇은 제품**이어야 한다. 학생부 진단은 온보딩과 개인화 입력으로 쓰고, 완성 문단 생성 기능은 MVP에서 제외한다.

### MVP 핵심 기능

| 우선순위 | 기능 | 내용 | 검증 지표 |
|---:|---|---|---|
| 1 | 수행평가 탐구 플래너 | 막연한 수행평가를 자료 조사, 교과 개념 연결, 탐구 질문, 개요, 발표 준비로 분해 | 첫 TODO 생성률, 체크 완료율 |
| 2 | 탐구 질문 만들기 | 교과 단원과 학생 관심사를 바탕으로 수업 중 활용 가능한 질문 생성 | 질문 저장률, 수업/보고서 활용률 |
| 3 | 지식 확장형 꼬리 질문 로드맵 | 완료한 활동에서 다음 탐구 주제와 후속 행동 추천 | 후속 TODO 실행률, 2회차 재방문율 |
| 4 | 학생부 진단 Lite | 학생부/활동 기록에서 강점, 약점, 부족 근거를 표시해 개인화 입력으로 활용 | 무료 진단 완료율, 개인화 동의율 |
| 5 | 월말 성장 리캡 | 완료 활동, 생성 질문, 탐구 흐름을 학부모용 정적 리포트로 요약 | 학부모 열람률, 구독 유지 의향 |
| 6 | 학종 부합도 진단 | 학생이 직접 쓴 초안의 누락 요소와 논리적 비약만 경고 | 재작성 요구율, 대필 오해 민원 0건 |

### 제한 파일럿 기능

| 기능 | 넣는 방식 | 주의점 |
|---|---|---|
| 미래 생기부 미리보기 | 로드맵을 따르면 어떤 역량 근거가 쌓이는지 시각화 | 공식 NEIS 기록 문장처럼 보이면 안 됨 |
| 빈틈 전형 후보 | 지원 자격 체크와 준비 로드맵으로 제한 | “쉬운 전형” 또는 “합격 가능성 높음” 표현 금지 |
| 검증 멘토 피드백 | 5~10명의 수동 검증 멘토가 30분 화상 피드백 | 전국 매칭 플랫폼으로 시작하지 않음 |
| 메디컬/예체능 vertical | 일부 케이스 인터뷰와 컨시어지 실험 | 전문가 검수 전 자동 판정 금지 |

### MVP에서 제외하거나 강하게 제한할 기능

| 기능 | 판단 | 이유 |
|---|---|---|
| 완성 문단 생성 | 제외 | 생성형 AI 결과물을 그대로 제출하거나 학생부에 반영하는 대필 오해 위험이 큼 |
| 문장 단위 첨삭 | 제한 | 학생 문장을 AI가 재작성하면 첨삭·대필 논란으로 이어질 수 있음 |
| 세특 문장 미리보기 | 제한 | 공식 기록 생성처럼 보이지 않도록 역량 근거와 활동 로그 중심으로 표현 |
| 합격 가능성/합격률 | 제외 | 데이터 신뢰와 표시광고 리스크가 큼 |

## 5. 가격과 수익화 가설

| 단계 | 가격 가설 | 제공 가치 | 목적 |
|---|---:|---|---|
| 무료 탐구 진단 Lite | 0원 | 수행평가 입력, 첫 탐구 질문 3개, 첫 TODO 일부 | 리드 수집과 즉시 효용 검증 |
| 1회 탐구 플래너 | 29,000~39,000원 | 수행평가 TODO 전체, 탐구 질문, 개요, 후속 로드맵 | 첫 결제 허들 검증 |
| 월간 AI 코치 | 월 49,000~99,000원 | 매월 수행평가/활동 업데이트, 꼬리 질문 로드맵, 월말 리캡 | 장기 사용 검증 |
| AI+전공 멘토 PM | 월 200,000~400,000원 | 격주/주간 멘토 체크, 탐구 실행 관리, 면접 피드백 | 대치동 고가 컨설팅과 AI 리포트 사이의 중간 시장 검증 |
| 전문가 검수 옵션 | 건당/월 별도 | 고위험 문구, 전형 전략, 법무·윤리 표현 검토 | 신뢰와 리스크 관리 |

❓ 가격은 실제 결제 데이터가 없으므로 가설이다. 멘토 미팅에서는 `3만 원대 리포트부터 검증할지`, `월간 코치로 바로 갈지`, `AI+전공 멘토 PM이 중산층 학부모에게 수용 가능한지`를 확인해야 한다.

## 6. 데이터·법무·운영 원칙

### 결론

MIRI는 학생 데이터를 회사가 오래 쌓아두는 서비스가 아니라, **학생/학부모가 업로드·삭제·내보내기·공유 권한을 통제하는 학생 데이터 Vault**로 설계해야 한다.

| 원칙 | 실행 방안 |
|---|---|
| 최소 수집 | MVP에서는 학생부/활동 기록 중심으로 받고, 사회통합·건강·장애·가족관계 증빙 원본은 받지 않음 |
| 분석 후 삭제 | 원본 학생부는 분석 후 삭제하고 결과 리포트만 선택 보관 |
| 권한 통제 | 학생/학부모가 멘토·컨설턴트에게 공유 범위를 선택 |
| 근거 표시 | 모든 추천은 수업 단원, 학생 입력, 완료 활동, 공개 모집요강 근거와 연결 |
| 자동화 고지 | AI 결과는 참고용이며 최종 판단은 학생·학부모·전문가가 한다고 표시 |
| 대필 오해 방지 | `NEIS 공식 생기부 생성`, `세특 대필`, `합격 보장` 표현 금지 |

### 규제 친화 UX 원칙

| 원칙 | 구현 |
|---|---|
| No Prose Generation | 완성 문단 대신 불릿, 개요표, 질문, 행동 체크리스트로 출력 |
| Cognitive Friction | 학생이 자기 생각, 키워드, 자료 링크를 입력해야 다음 단계가 열리도록 설계 |
| Why This Task | 추천 TODO마다 “왜 지금 이 행동을 추천하는지” 근거를 표시 |
| Copy-Paste 방지 | 제출용 문장 생성 버튼을 두지 않고 조사하기, 질문 고르기, 개요 만들기 버튼 중심으로 구성 |
| 교과 본질 우선 | 모든 과목을 억지로 진로와 연결하지 않고, 과목 고유 역량을 먼저 반영 |

### 금지하거나 뒤로 미룰 기능

- 합격 확률/합격 가능성 단정
- 공식 NEIS 생기부 생성처럼 보이는 기능
- 근거 없는 활동·수상·실적 생성
- 대학별 내부 평가 기준을 아는 것처럼 단정
- 미성년자 개인정보를 동의 없이 장기 저장
- 전국 멘토 매칭 플랫폼을 MVP 본체로 바로 구축
- 수행평가나 보고서 문장을 AI가 직접 고쳐주는 기능을 기본값으로 제공

## 7. 확장 전략

### 7.1 빈틈 전형

빈틈 전형은 `쉬운 전형 추천`이 아니라 **지원 자격을 놓치기 쉬운 정원외·특수전형을 구조화하는 기능**으로 잡아야 한다.

| 범위 | 접근 |
|---|---|
| 초기 DB | 대학 30~50개, 전형군 20개, 자격조건 체크리스트 |
| 핵심 기능 | 전형 후보 발굴, 지원 자격 체크, 준비 서류, 마감일 |
| 피해야 할 표현 | “합격 가능성이 높다”, “아는 사람만 아는 쉬운 전형” |
| 검증 방식 | 모집요강 기반 체크리스트 정확도와 사용자 인터뷰 |

### 7.2 메디컬 vertical

메디컬은 지불 의향이 높을 가능성이 큰 프리미엄 후보지만, 합격 확률과 내부 평가 기준 단정은 피해야 한다.

| 기능 | 설명 |
|---|---|
| 생기부 무결성 진단 | 의학 계열 희망과 현재 활동의 연결성, 누락 근거 점검 |
| Deep Dive 트리 | 의학 주제 탐구가 얕게 끝나지 않도록 질문 구조화 |
| MMI 스파링 | 딜레마 질문, 꼬리질문, 논리 구조 점검 |
| 대학별 공개 평가 렌즈 | 공개 모집요강과 인재상 기반 체크리스트 |

### 7.3 예체능·체대 vertical

예체능은 AI가 실기를 판정하는 방향보다 **서류·면접·환산 계산·일정 관리 보조**가 안전하다.

| 분야 | MIRI가 들어갈 수 있는 기능 |
|---|---|
| 미대/예술 | 미술활동보고서, 작품 설명, 면접 질문 |
| 영화/영상/한예종 | 시놉시스 구조 점검, 제시문 면접 스파링 |
| 체대 | 실기·내신·수능 환산 체크, 목표 기록 역산 |
| 음대/공연 | 레슨/콩쿠르/실기 일정 정리, 면접 대비 |

### 7.4 특목고·고입 버전

특목고 정보는 수요가 있지만 현재 MIRI의 메인 타겟인 `고등학생 학종 코치`와 바로 붙이면 타겟이 흔들릴 수 있다. V2에서 `고입 버전 MIRI` 또는 `중3 진로·고입 로드맵`으로 분리하는 편이 안전하다.

## 8. 발표용 핵심 문장

- “미리의 빈 공간은 생기부 AI 리포트가 아니라, 수행평가와 교과 탐구를 학생이 직접 설계하도록 돕는 AI 학종 프로젝트 관리입니다.”
- “대형 입시 브랜드는 원서와 합격예측에 강하고, 과외 플랫폼은 교과 수업에 강합니다. 미리는 그 사이에서 학생부 근거와 이번 주 행동을 연결합니다.”
- “우리는 합격 확률을 팔지 않고, 준비 상태·누락 항목·다음 행동을 팝니다.”
- “AI 출력은 완성 문단이 아니라 질문, 개요, 체크리스트, 추천 근거로 제한합니다.”
- “MVP는 전국 멘토 매칭 플랫폼이 아니라 학생부 진단 리포트와 수동 멘토 피드백을 묶은 컨시어지 실험으로 시작합니다.”
- “미래 생기부 미리보기는 공식 기록 생성이 아니라 로드맵 결과를 이해시키는 참고 화면으로 제한합니다.”

## 9. 멘토에게 물어볼 것

1. ❓ 첫 유료 상품은 `3만 원대 수행평가 탐구 플래너`가 맞는가, 아니면 `월 5만~10만 원 AI 코치`가 맞는가?
2. ❓ MIRI의 첫 포지션은 `AI 학생부 리포트`보다 `수행평가·교과 탐구 플래너`가 더 설득력 있는가?
3. ❓ 완성 문단 생성 없이 `질문/개요/TODO/추천 근거`만으로 학부모 결제가 가능한가?
4. ❓ `학종 부합도 진단`은 MVP에 넣을지, 대필 오해를 피하기 위해 V2로 미룰지?
5. ❓ 빈틈 전형 매칭은 MVP에 넣을지, 학생부 코치 V2 차별화 기능으로 둘지?
6. ❓ AI기본법상 교육 분야 고영향 AI 오해를 피하려면 어떤 표현과 고지 문구가 필요한가?

## 10. 최종 실행안

| 기간 | 할 일 | 산출물 |
|---|---|---|
| 1~2주 | 수행평가 입력 → TODO 분해 프로토타입 제작 | 수행평가 탐구 플래너 샘플 10개 |
| 3~4주 | 탐구 질문 만들기와 추천 근거 UI 제작 | 질문 카드, Why This Task 예시 |
| 5~6주 | 학생 10명, 학부모 10명에게 문구/기능 검증 | 결제 의향, 대필 오해 지점, 카피 피드백 |
| 7~8주 | 지식 확장형 꼬리 질문 로드맵 제작 | 완료 활동 → 후속 TODO 연결 샘플 |
| 9~12주 | 무료 탐구 진단 Lite 랜딩과 카카오 채널 퍼널 | 리드 수집, 첫 TODO 완료율, 전환율 |
| 13~24주 | 월간 AI 코치와 월말 성장 리캡 검증 | 월 구독 유지율, 실행률, 학부모 열람률 |
