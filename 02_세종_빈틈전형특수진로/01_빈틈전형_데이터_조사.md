# 빈틈 전형 데이터 조사

## 한 줄 결론

빈틈 전형은 “꿀전형”이 아니라 **자격조건이 복잡해서 학생이 놓치기 쉬운 전형**이다. MVP는 합격 예측보다 `지원 자격 체크리스트`, `전형 후보 추천`, `준비 서류/일정 로드맵`으로 시작하는 편이 안전하다.

## 데이터 출처

| 출처 | 얻을 수 있는 데이터 | 활용 방식 |
|---|---|---|
| 대교협 대학입학전형시행계획 | 전국 대학의 전형 계획, 모집인원, 전형 구조 | 기준 데이터로 사용 |
| 대교협 대학입학전형기본사항 | 전형 수립 원칙, 모집시기, 특별전형 기준 | 법적/제도적 기준 체크 |
| 대입정보포털 어디가 | 대학별 전형정보 검색 | 사용자 검색 UX 벤치마크 |
| 대학별 입학처 PDF | 최종 모집요강, 자격조건, 제출서류 | 실제 DB 원천 |
| KUSF 체육특기자대입포털 | 체육특기자 전형 요강·설명회·자료 | 체육특기 vertical |
| 고입정보포털 | 특목고/자사고 학교별 요강 | 고입 vertical |

## MVP 기능으로 바꾸면

1. 학생 정보 입력: 지역, 고교 유형, 재학 이력, 가족/사회통합 자격, 특기
2. 자격 조건 체크: 해당 가능 전형군 표시
3. 대학별 후보: 조건에 맞을 수 있는 전형 후보 리스트
4. 리스크 표시: “증빙 필요”, “최종 모집요강 확인 필요”, “전문가 검토 필요”
5. 준비 로드맵: 서류, 일정, 학생부 보완 포인트

## 가져가면 좋은 메시지

- “숨은 합격 전형을 보장합니다”가 아니라 “놓치기 쉬운 지원 자격을 먼저 확인합니다.”
- “AI가 대학을 찍어줍니다”가 아니라 “공식 요강 근거로 전형 후보를 좁혀줍니다.”

## ❓ 멘토에게 검증받을 것

- ❓ 전형군 20개를 모두 넣는 것보다 농어촌/고른기회/특성화/재외국민부터 좁히는 게 나은가?
- ❓ 대학별 PDF 파싱을 자동화해도 법무/저작권/운영 리스크가 괜찮은가?
- ❓ “빈틈 전형”이라는 표현이 마케팅에 적합한가, 아니면 “맞춤 전형 탐색”이 안전한가?
