## 역할

- `01_서비스_정의`: 미리(MIRI)가 누구의 어떤 문제를 푸는 서비스인지 정의한다.
- `02_브랜드_커뮤니케이션`: 외부에 어떤 이름과 문장으로 말할지 관리한다.
- `03_서비스_작동구조`: 온보딩부터 액션 추천, 아카이빙, 갱신까지 제품이 어떻게 작동하는지 정리한다.

## 중복 정리 원칙

- 서비스의 한 줄 정의와 범위는 `01_서비스_정의`에만 둔다.
- 브랜드명, 핵심 메시지, 표현 원칙은 `02_브랜드_커뮤니케이션`에만 둔다.
- 작동 사이클, MVP 기능 범위, 반복 사용 구조는 `03_서비스_작동구조`에만 둔다.
