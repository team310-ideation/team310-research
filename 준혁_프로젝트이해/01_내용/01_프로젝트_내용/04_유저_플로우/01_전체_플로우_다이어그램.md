# 전체 유저 플로우 다이어그램

## 핵심 흐름

```mermaid
flowchart TD
  A["유입<br/>광고·커뮤니티·지인 추천"] --> B["랜딩 페이지<br/>고3까지 학교생활·생기부 설계"]
  B --> C{"사용자 선택"}

  C --> S1["학생<br/>무료 미리보기 시작"]
  C --> P1["학부모<br/>자녀 결과 확인 목적"]

  S1 --> S2["기본 정보 입력<br/>학년·학교 유형·관심 진로"]
  S2 --> S3["기존 생기부·활동 자료 입력"]
  S3 --> S4["AI 진로 인터뷰"]
  S4 --> R1["미래 생기부 미리보기 생성"]

  P1 --> P2["문제 공감<br/>고교학점제·과목 선택·생기부 불안"]
  P2 --> P3["샘플 결과물 확인<br/>미래 생기부·학교생활 로드맵"]
  P3 --> P4["자녀와 함께<br/>무료 미리보기 진행"]
  P4 --> S2

  R1 --> R2["강점·약점 요약"]
  R1 --> R3["진로 후보와<br/>고3까지 학교생활 로드맵"]
  R1 --> R4["하루/주간/월간/학기 액션"]

  R2 --> A1["첫 액션 선택"]
  R3 --> A1
  R4 --> A1

  A1 --> D1["활동 아카이빙<br/>과목·수행평가·독서·동아리·탐구"]
  D1 --> D2["AI 분류<br/>과목·진로 키워드·생기부 항목"]
  D2 --> D3["리포트 갱신<br/>갭 분석·다음 액션"]
  D3 --> A2["다음 선택 실행<br/>과목·활동·탐구·학교생활"]
  A2 --> D1

  R1 --> P5["학부모 공유"]
  P5 --> P6["월간 요약·진행률 확인"]
  P6 --> P7["사전예약·출시 알림·결제 의향 확인"]

  R1 --> V1["베타 신청 또는 사전예약"]
  V1 --> V2["후속 알림<br/>이메일·카톡 채널"]
```

## 플로우 원칙

- 첫 경험은 회원가입보다 결과물 확인을 우선한다.
- 학생에게는 "지금 학교생활에서 무엇을 선택해야 하는가"를 보여준다.
- 학부모에게는 "우리 아이가 고3까지 어떤 방향으로 가고 있는가"를 요약한다.
- 반복 사용의 중심은 활동 아카이빙과 리포트 갱신이다.
