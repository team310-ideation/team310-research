# 8개월 아카이빙 플로우

## 핵심 질문

- 8개월 동안 사용자를 붙잡을 수 있는 기능이 있는가?
- 공개된 범용 AI와 다른 차별점은 무엇인가?
- 월 3만 원을 내는 이유가 단발 리포트가 아니라 누적 아카이빙에서 생기는가?

## 플로우 초안

| 단계 | 기능 | 사용자 가치 |
| --- | --- | --- |
| 1개월차 | 생기부/활동 분석 리포트 | 현재 위치 파악 |
| 2개월차 | 과목·활동 체크리스트 | 해야 할 행동 구체화 |
| 3~4개월차 | 수행평가/탐구 기록 누적 | 활동 근거 축적 |
| 5~6개월차 | 리포트 갱신과 약점 보완 | 방향 수정 |
| 7개월차 | 면접 질문·스토리 정리 | 설명 가능성 강화 |
| 8개월차 | 다음 학기/진로 로드맵 | 장기 전략 연결 |

## 아카이빙 원칙

- 학생이 입력한 활동, 체크리스트, 리포트 결과를 시간순으로 누적한다.
- 학생이 자기 데이터를 소유하고 내보낼 수 있어야 한다.
- 멘토가 바뀌어도 기록이 남아 전체 맥락을 잃지 않아야 한다.
