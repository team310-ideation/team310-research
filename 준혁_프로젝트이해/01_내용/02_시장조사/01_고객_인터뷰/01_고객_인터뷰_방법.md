## 3. 고객 인터뷰 방법론 (핵심)

### 3.1 가설 기반 인터뷰

- **가설을 먼저 세우고 그것을 검증하는 형태로 진행**
- 예: "고교 진학을 원하는 중3 학부모는 고민이 많을 것이다"
- 100건 인터뷰 목표 후 가설 진위 판단 → 다음 가설로 진도

### 3.2 금기 사항 — 유도 신문 금지

- 가설을 직접적으로 물으면 당연히 그렇다는 답이 나옴 → 무의미
- 일상적 질문으로 시작해 단계별로 본질에 접근
- "고민이 있으세요?" → "어떤 고민이세요?" 순서로 자연스럽게 유도

### 3.3 솔루션 제시 금지

- "이렇게 하면 해결될까요?" 류의 질문 금지
- 인터뷰어 앞에서 대면으로 "안 좋을 것 같다"고 답할 사람은 드묾
- 온라인(익명) vs 오프라인 인터뷰 결과 차이 큼
- 가격 의사 확인 예시: "좋을 것 같다"(긍정) → "10만 원에 사겠냐"(진짜 반응)

### 3.4 설문조사의 한계

- 논문에서도 인터뷰·설문 기반은 지적 대상
- 질문 설계에 상당한 시간과 전문성 필요
- 박사 논문 수준에서는 순수 설문만으로는 인정받기 어려움

### 3.5 인터뷰 수집 시 필수 항목

- 인구통계 정보(지역·나이대·성별) 선제 수집
- 누적 데이터로 통계 분석 가능하도록 설계

### 인터뷰 체크리스트

- [ ] 가설을 먼저 쓴다.
- [ ] 가설을 직접 물어보지 않는다.
- [ ] 솔루션을 먼저 설명하지 않는다.
- [ ] 지역·나이대·성별을 먼저 수집한다.
- [ ] 긍정 반응 뒤에는 가격 질문으로 진짜 반응을 확인한다.
- [ ] 온라인 익명 조사와 오프라인 인터뷰 결과를 구분한다.

## 6. 내일 인터뷰 준비 논의

### 6.1 인터뷰 목적 설정

- **가설 검증형 인터뷰**로 진행
- 주요 가설 예시:
    1. 중3 자녀 학부모는 고교학점제에 대해 고민이 많을 것이다
    2. 학부모들은 이미 어떤 대책을 준비하고 있을 것이다
    3. 특정 소득·지역 계층이 더 관심이 많을 것이다 (반대 가설도 검증 필요)

### 6.2 실행 권장 사항

- 학부모처럼 대면 인터뷰 진행 (피칭 스타일 아님)
- 기본 인적 사항(지역·나이대) 사전 확보
- 거절 가능성 고려하여 톤 조정
- 인터뷰 데이터 누적 시 통계 활용 가능하도록 폼 설계
- 온라인 익명 조사(맘카페 등) 병행 검토

- 외국인 학부모 등 개인정보 민감 케이스는 별도 대응한다.

## 세종 서비스 서사 회의 반영

### 인터뷰 필요성 논쟁

- 세종 입장: 직관적으로 만들고 후기 기반 개선이 더 효율적이며, 인터뷰 비용이 부담된다.
- 세민 입장: 방향을 잘못 잡으면 회복 비용이 더 크므로 가설 검증이 필수다.
- 결론: 인터뷰는 진행하되 효율적으로 운영한다.

### 100건 목표 운영 방안

- 세그먼트별로 분배한다.

  - 중위권 30

- 준혁의 모교 선생님 네트워크를 활용한다.
- 화상 인터뷰와 커피 쿠폰 인센티브를 검토한다.
