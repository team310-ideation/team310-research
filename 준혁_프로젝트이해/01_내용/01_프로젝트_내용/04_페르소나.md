# 페르소나

## 왜 필요한가

- 팀원마다 서비스를 다르게 이해하는 것을 막기 위해 실제 사용자 1명을 구체적으로 잡는다.
- 기획서, 피칭, MVP 기능, 인터뷰 질문을 모두 같은 사용자 스토리라인으로 검증한다.

## 페르소나 후보 1: 분당 거주 중3 여학생

| 항목 | 내용 |
| --- | --- |
| 사용자 | 분당 거주 중3 여학생 |
| 상황 | 고교학점제 환경에서 고등학교 선택과 과목 선택, 생기부 방향 설계를 어려워함 |
| 보호자 | 진로와 입시 정보에는 관심이 있지만 대치동식 고가 컨설팅은 부담스러워함 |
| 핵심 문제 | "우리 아이가 앞으로 어떤 스토리로 고등학교 생활과 대학 진학까지 이어가야 하는지"가 불명확함 |
| 기대 가치 | 학생의 현재 관심사와 활동을 바탕으로 고등학교 생활 설계, 과목 선택, 활동 체크리스트를 구체화 |

## 페르소나 후보 2: 박세민형 학생

| 항목 | 내용 |
| --- | --- |
| 사용자 | 박세민 |
| 상황 | 평범한 학생이고 쿠키 만드는 것이 취미 |
| 문제 | 고교학점제 상황에서 어떤 과목과 활동 플로우를 짜야 하는지 모름 |
| 가치 | 자신만의 생기부 스토리를 만들 수 있도록 과목, 활동, 체크리스트를 제안 |
| 기능 후보 | 과목 추천, 초개인화 학원/선생님 추천, 학생용 체크리스트 |

## 검증 질문

- [ ] 중3 학부모가 실제로 이 정도의 위기감을 갖고 있는가?
- [ ] 중학생 타겟과 고등학생 타겟 중 6개월 동안 어느 쪽을 더 뾰족하게 잡을 것인가?
- [ ] 학생 본인보다 부모가 결제 의사결정자인 구조가 맞는가?

## 출처

- 조재홍 멘토링 회의록
- 사용자 추가 메모
