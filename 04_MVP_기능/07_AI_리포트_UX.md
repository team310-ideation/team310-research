# AI 리포트 UX

## 리포트 구조 개선

- 리포트 첫 단에 고등학교 추천이 바로 나오면 문맥상 뜬금없다.
- 배경지식 격차를 줄이기 위해 생기부 구성 소개를 앞단에 배치한다.
- "세특"이라는 줄임말 대신 "세부능력특기사항"으로 풀어서 설명한다.
- 생기부 면접과 제시문 면접을 구분한다.

## 점수와 캐릭터성

- 단순 웰빙/성향 지표보다 점수를 다양화한다.
- 너무 무난한 표현만 쓰면 기억에 남지 않으므로, 일부 극단적 표현을 섞어 명확한 캐릭터성을 만든다.
- 다만 공식 평가나 합격 보장처럼 보이는 표현은 피한다.

## 학생용 체크리스트

- 학생이 직접 실행할 수 있는 체크리스트를 리포트에 포함한다.
- 체크리스트는 과목 선택, 활동 설계, 탐구 주제, 면접 준비처럼 행동 단위로 쪼갠다.
- 학생 입력이 있어야 다음 단계가 활성화되는 구조를 유지한다.

## 리포트 앞단 구성 초안

1. 생기부 구성 설명
2. 학생 현재 기록 요약
3. 강점/공백 진단
4. 전공·진로 스토리 후보
5. 과목/활동 체크리스트
6. 면접 대비 질문
7. 고등학교 또는 대학/학과 추천 후보
