# 브랜드·커뮤니케이션 기준

## 브랜드 기준

- 팀 프로젝트명은 **미리(MIRI)** 로 통일한다.
- 외부 커뮤니케이션에서는 `미리(MIRI)`만 현재 프로젝트명으로 사용한다.
- `휴포지토리`는 상위 비전 또는 기술적 배경으로만 설명한다.
- `찐로`는 현재 진행 프로젝트가 아니라, 미리 이후 확장 가능한 진로·인생설계 프로젝트로 보관한다.

## 브랜드 구조

| 구분 | 현재 정리 |
| --- | --- |
| 현재 프로젝트명 | 미리(MIRI) |
| 상위 배경 | 휴포지토리: 개인 기록 + AI 맥락 업데이트 |
| 현재 서비스 영역 | 청소년 진로·생기부·학교생활 의사결정 OS |
| 현재 집중 | 중3~고3 학생과 학부모 대상 서비스 |
| 확장 후보 | 찐로: 미리 이후 진로·커리어·인생설계 영역으로 확장 가능한 프로젝트 |

## 핵심 메시지

- "생기부를 대신 써주는 서비스가 아니라, 학생의 선택을 설계하는 서비스다."
- "고교학점제 시대에 학생과 학부모가 과목·활동·탐구 선택과 학교생활 설계를 혼자 감당하지 않게 한다."
- "단발성 진단이 아니라, 중3부터 고3까지 학생 데이터가 쌓일수록 다음 선택이 정교해지는 서비스다."

## 표현 원칙

| 쓰는 표현 | 피하는 표현 |
| --- | --- |
| 학교생활 선택을 설계한다 | 생기부를 대신 작성한다 |
| 미래 생기부 미리보기 | 합격을 예측한다 |
| 다음 액션을 추천한다 | 정답을 정해준다 |
| 학생 데이터 기반 의사결정 | 범용 AI 상담 |
| 중3~고3 학교생활 로드맵 | 단발성 진로 검사 |

## 찐로 처리 기준

- 찐로는 현재 제품명이 아니다.
- 찐로는 미리에서 쌓인 학생 데이터가 입시 이후 진로·커리어 의사결정으로 확장될 때 검토한다.
- 외부 발표에서는 미리와 찐로를 병렬 브랜드처럼 설명하지 않는다.
- 찐로 관련 자세한 내용은 [찐로 확장 아이디어](../06_비전_확장/02_찐로_확장_아이디어.md)에서 관리한다.
