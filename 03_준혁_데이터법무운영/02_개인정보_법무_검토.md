# 미성년자 개인정보 법무 검토

## 한 줄 결론

미리(MIRI)는 고등학생 대상 서비스라도 **미성년자 학생부·활동기록·진로정보를 처리하는 서비스**이므로 개인정보보호법상 최소수집, 명확한 동의, 분석 후 삭제, 자동화된 결정 고지, 민감정보 회피가 필수다. 특히 만 14세 미만 개인정보는 단순히 고지·명시만으로 책임을 피할 수 없으며, 법정대리인 동의가 없는 경우 수집 자체가 불법이 될 수 있다.

## 우리가 처리하려는 데이터

| 데이터 | 예시 | 위험도 | MVP 처리 방침 |
|---|---|---:|---|
| 기본 학생 정보 | 이름, 학년, 학교, 희망학과 | 중간 | 이름/학교는 선택값 또는 마스킹 |
| 학생부 원문 | 세특, 창체, 행특, 진로활동 | 높음 | 분석 후 원본 삭제 |
| 활동 기록 | 보고서, 독서, 탐구, 동아리 | 중간 | 결과 근거로만 저장 |
| 상담 기록 | 학부모/컨설턴트 메모 | 높음 | MVP에서는 받지 않거나 별도 동의 |
| 사회통합 자격 | 기초생활, 차상위, 장애, 가족상황 등 | 매우 높음 | MVP에서 원본 증빙 수집 금지 |
| AI 분석 결과 | 진단, 적합도, 로드맵, 면접 질문 | 중간 | 참고용, 사람이 최종 판단 |

## 핵심 법무 체크

### 1. 만 14세 미만 아동

개인정보 보호법 제22조의2는 만 14세 미만 아동의 개인정보 처리에 동의가 필요한 경우 법정대리인의 동의를 요구한다. 시행령 제17조의2는 법정대리인 동의 확인 방법을 다룬다.

### 미리 적용

- MVP 타겟은 고1~고3으로 두는 편이 안전하다.
- 중학생/특목고 타겟까지 확장하면 법정대리인 동의 UI가 필요하다.
- 법정대리인 동의 확인을 위해 필요한 최소정보만 받아야 한다.
- 단순히 "중3 이상만 사용"이라는 문구를 붙이는 것만으로는 법적 책임을 회피할 수 없다. 실제로 13세 사용자가 데이터를 올리고 회사가 저장하면 불법 수집 방치로 책임이 발생할 수 있다.

### 1-1. MVP 단계 현실적 방어막

1. Age Gate 체크박스
   - 업로드 직전 필수 확인 항목으로 "본인은 만 14세 이상이거나, 만 14세 미만 아동의 데이터를 제공하는 법정대리인(부모)입니다"를 추가한다.
   - 이 체크박스는 명시적 확인을 받았다는 최소한의 방어 논리를 제공한다.
2. 학부모 타겟팅
   - 초기 마케팅과 랜딩 페이지는 '학부모 대상 서비스'로 명확히 포장한다.
   - 부모가 자녀 데이터를 제출하면 법정대리인 동의가 있는 것으로 볼 수 있어 법적 리스크가 낮아진다.
3. 데이터를 저장하지 않는 구조
   - 학생부 원본은 AI 분석 완료 후 즉시 영구 삭제한다.
   - 서버에 장기 저장하지 않는 구조를 MVP에서 우선 적용하면 가장 낮은 리스크가 된다.

### 2. 개인정보 수집·이용 동의

학생부 파일은 단순 텍스트가 아니라 학생의 학업, 활동, 평가, 진로가 결합된 개인정보다. 따라서 수집 목적, 항목, 보유기간, 동의 거부권을 명확히 알려야 한다.

### 동의 문구 초안

```text
미리(MIRI)는 학생부종합전형 대비 참고 진단을 제공하기 위해 사용자가 업로드한 학생부/활동기록 파일을 일시적으로 처리합니다.
업로드 원본 파일은 분석 완료 후 즉시 삭제되며, 사용자가 저장을 선택한 경우 분석 결과만 계정에 보관됩니다.
본 서비스의 결과는 공식 입시 평가나 합격 가능성 보장이 아니며, 학생·학부모·전문가 검토를 위한 참고자료입니다.
```

### 3. 민감정보와 고유식별정보

개인정보 보호법 제23조는 민감정보 처리 제한을 둔다. 학생부 자체가 법상 민감정보 전체에 해당한다고 단정할 수는 없지만, 빈틈 전형/고른기회/특수교육/건강/가족상황 증빙이 들어오면 민감정보 또는 고위험 개인정보가 섞일 수 있다.

### MVP 금지 항목

- 주민등록번호
- 건강/장애/진단서 원본
- 기초생활수급/차상위 증빙 원본
- 가족관계증명서 원본
- 여권/외국인등록증
- 학교폭력/징계 상세 자료

### 대체 방식

증빙 원본 대신 체크박스 방식으로 받는다.

```text
농어촌학생 해당 가능성 있음
기초생활/차상위 해당 가능성 있음
특성화고 동일계열 확인 필요
재외국민/외국인 전형 확인 필요
```

### 4. 파기 의무

개인정보 보호법 제21조는 보유기간 경과, 처리 목적 달성 등 개인정보가 불필요하게 되었을 때 파기해야 한다는 취지의 조항이다.

### 미리 적용

- 업로드 원본: 분석 완료 즉시 삭제
- 분석 결과: 사용자가 저장 선택 시 보관
- 비회원 무료진단: 세션 종료 또는 24시간 내 삭제
- 공유 링크: 기본 7일 만료
- 탈퇴 시: 계정 및 분석 결과 삭제
- MVP 테스트 단계에서는 원본 학생부를 서버에 저장하지 않는 구조가 가장 낮은 리스크다. AI 분석 후 즉시 영구 파기하는 절차를 설계해야 한다.

### 5. 자동화된 결정

개인정보 보호법 제37조의2는 AI 등 완전히 자동화된 시스템으로 이루어진 결정이 정보주체의 권리 또는 의무에 중대한 영향을 미치는 경우 거부·설명 요구권을 규정한다.

### 미리 적용

미리는 다음 표현을 피해야 한다.

- “이 학생은 불합격 가능성이 높음”
- “이 전형 지원 불가”
- “AI 공식 학생부 평가”
- “이 로드맵을 하면 합격 가능”

대신 다음처럼 표현한다.

- “지원 가능성을 검토해볼 후보”
- “추가 확인 필요”
- “공식 모집요강 확인 필요”
- “학생부종합전형 대비 참고 진단”

### 6. 생성형 AI 개인정보 처리

개인정보보호위원회는 생성형 AI 개발·활용을 위한 개인정보 처리 안내서를 공개했다. 생성형 AI 수명주기 전반에서 개인정보 처리 근거, 안전조치, 프라이버시 리스크 관리를 요구하는 방향이다.

### 미리 적용

- 외부 LLM API에 학생부 원문을 보낼 경우 처리위탁/국외이전/보관 여부 확인 필요
- 프롬프트에 이름·학교·전화번호를 보내지 않도록 사전 마스킹
- 결과물에 원문 개인정보가 재노출되는지 검사
- 모델 학습에 학생 데이터를 사용하지 않는다는 설정/계약 확인

### 7. AI기본법

인공지능 발전과 신뢰 기반 조성 등에 관한 기본법은 2026년 1월 22일부터 시행된다. 교육 영역은 고영향 AI 논의와 맞물릴 수 있으므로, 미리는 “사람이 최종 판단하는 참고용 코치”로 설계해야 한다.

### 고지 문구 초안

```text
본 결과는 생성형 AI가 사용자 제공 자료를 바탕으로 생성한 참고용 분석입니다.
대학 지원 가능 여부, 합격 가능성, 학교생활기록부 공식 기재 가능성을 보장하지 않습니다.
최종 판단은 학생·학부모·교사·입시 전문가가 공식 모집요강과 학교 지침을 확인해 진행해야 합니다.
```

## 우리 서비스가 가장 주의해야 할 5가지 리스크

| 리스크 | 왜 위험한가 | 대응 |
|---|---|---|
| 원본 학생부 장기 저장 | 미성년자 학업·평가 정보 축적 | 분석 후 삭제 기본값 |
| 만 14세 미만 동의 누락 | 법정대리인 동의 이슈 | MVP는 고1~고3 우선 |
| 민감 증빙 수집 | 사회통합/건강/장애 정보 포함 가능 | 체크박스만 받고 원본 금지 |
| 합격 가능성 단정 | 자동화된 결정/오인 리스크 | 리스크 분석·후보 표현 |
| 외부 LLM 전송 | 국외이전/학습/보관 불명확 | 마스킹, DPA, 저장 금지 확인 |

## 개인정보 처리방침에 들어갈 최소 항목

1. 처리 목적: 학생부종합전형 대비 참고 진단
2. 처리 항목: 업로드 파일, 희망학과, 학년, 분석 결과
3. 보유 기간: 원본 즉시 삭제, 결과는 회원 선택 보관
4. 제3자 제공: 없음 또는 명시
5. 처리위탁: LLM/API/클라우드 사업자 명시
6. 국외 이전: 외부 API가 해외 서버라면 별도 고지
7. 정보주체 권리: 열람, 정정, 삭제, 처리정지
8. 만 14세 미만: 법정대리인 동의 절차
9. 자동화된 결정: 참고용이며 최종 판단 아님

## 멘토/변호사에게 검증받을 것

- ❓ 학생부 원본을 외부 LLM API에 전송해도 되는지, 어떤 위탁/국외이전 고지가 필요한지?
- ❓ 학생부 AI 진단이 AI기본법상 고영향 AI로 오인될 가능성은 어느 정도인지?
- ❓ 고른기회/사회통합 전형 추천을 위해 자격 체크박스를 받는 것까지 가능한지?
- ❓ 만 14세 이상 고등학생의 경우 학부모 동의 없이 학생 본인 동의만으로 충분한지, 유료 결제 주체가 학부모이면 별도 동의가 필요한지?
