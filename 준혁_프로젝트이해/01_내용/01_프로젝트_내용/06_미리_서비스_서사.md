# 미리 서비스 서사

## 브랜드 구조

| 구분 | 내용 |
| --- | --- |
| 상위 브랜드 | 휴포지토리 |
| 하위 브랜드 | 미리(MIRI): 청소년 생기부 기반 서비스 |
| 향후 확장 | 찐로 등 성인용 하위 브랜드 가능 |
| 운영 방식 | 하위 브랜드별 오너십 분리 운영, CEO 체제 가능성 |

## 핵심 가치

> 초중 생기부와 모범 생기부 데이터를 바탕으로, 좋은 고등학교 생기부를 생성·관리하고 3년간 액션 아이템을 제시한다.

## 작동 사이클

1. 온보딩: 초중 생기부와 AI 인터뷰 답변 입력
2. 초기 데이터 생성: 학과·진로 적합도 분석, 강점/약점 도출
3. 3년 플랜 제시: 동아리, 도서, 대회, 세부능력특기사항 단위 추천
4. 실행 후 데이터 누적: 수행평가, 레포트, 발표 자료 아카이빙
5. 다음 액션 추천: 이전 데이터 맥락 기반 액션 아이템 제시
6. 고3 시점: 목표 학과 진학 가능한 생기부 완성 방향으로 업데이트

## 멘토링 중 정리된 한 문장 구조

```text
초중 생기부(사용자 데이터) + 모범 생기부(참조 데이터)
      ↓
AI 생성·분석
      ↓
3년 풀 관리 액션 아이템
      ↓
사용자 실행 + 데이터 누적
      ↓
업데이트된 생기부 및 TODO
```

## MVP 우선순위

### Must Have

- 초중 생기부 입력 → 미래 생기부 및 TODO 생성
- 3년 액션 아이템 제시
- 사용자 데이터 아카이빙
- 모범 생기부 RAG 데이터셋 구축

### Nice to Have

- 멘토 매칭
- 프롬프트 교육 및 스킬 공유 커뮤니티

## 서비스 정체성 메모

- 미리는 단순 "분석 후 추천"이 아니라 "3년 풀 관리"를 지향한다.
- 시장도 분석형 진단에서 풀세팅·전형 풀관리·심층 풀관리 등 장기 관리 패키지로 이동 중이다.
- 핵심 차별화는 데이터 아카이빙, AI 활용 능력 교육, 모범 생기부 데이터셋 확보에 있다.
