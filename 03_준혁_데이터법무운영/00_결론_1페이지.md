# 준혁 트랙 결론 1페이지: 데이터·법무·운영

> 미리(MIRI)는 “입시 데이터를 회사가 쌓는 서비스”가 아니라 **학생이 자기 데이터를 소유하고, AI가 근거 기반으로 진단·로드맵·면접 대비를 도와주는 데이터 지갑형 입시 코치**로 가야 한다.

## 핵심 발견 3가지

1. 금융/의료 마이데이터와 Solid 사례의 공통점은 `동의`, `전송`, `권한`, `이력`, `삭제`다. 미리도 학생 데이터 Vault를 핵심 신뢰 장치로 가져가야 한다.
2. 미성년자 학생부 데이터는 법무 리스크가 크다. 원본 학생부는 분석 후 삭제하고, 사회통합/건강/장애 등 고위험 증빙 원본은 MVP에서 받지 않는 편이 안전하다.
3. 마케팅은 맘카페/네이버로 리드를 만들고, 카카오톡 채널로 리텐션을 잡은 뒤, 무료 진단 Lite에서 3만 원대 전략 리포트로 전환하는 구조가 현실적이다.

## 우리가 가져갈 수 있는 것

- 학생 데이터 Vault: 업로드, 분석, 내보내기, 삭제, 공유 권한을 학생/학부모가 통제한다.
- 분석 후 삭제: 초기에는 원본 학생부를 장기 저장하지 않고 분석 결과만 선택 보관한다.
- MVP 방어막: `만 14세 이상/부모 확인` 체크박스 + `분석 후 즉시 파기` 명시.
- 무료→유료 전환: 무료 진단 Lite → 29,000~39,000원 전략 리포트 → 99,000원 이상 코치 패키지.
- 학부모 퍼널: 맘카페/네이버 콘텐츠 → 카카오톡 채널 → 리포트 결제 → 컨설턴트 검수.

## 우리가 가져갈 수 없는 것

- “합격 가능성” 단정: 데이터와 법무 리스크가 크다.
- “공식 생기부/NEIS 생성”: 학교 공식 기록처럼 오인될 수 있다.
- 미성년자 개인정보 무동의 저장: 만 14세 미만은 법정대리인 동의 이슈가 있고, 고등학생도 최소 수집·삭제 정책이 필요하다.
- 사회통합/건강/장애/가족관계 증빙 원본 수집: MVP에서는 체크박스로만 받는다.

## 출시 우선순위

1. 지금 넣기: 업로드, 정규식 파싱, 학생부 진단, 전공 적합도, 성장 로드맵, 면접 질문
2. 제한 파일럿: 빈틈 전형 후보, 미래 생기부 미리보기
3. 운영 필수: 원본 즉시 삭제, 결과 내보내기, 접근 로그, 카카오톡 채널 리텐션, 만 14세 이상/법정대리인 확인 체크박스
4. V2: 특수 진로, 특목고, 학부모 대시보드, 데이터 공유 권한 관리
5. 제외: 합격 가능성 시뮬레이션

## 멘토에게 물어볼 것

- `학생부 진단 + 로드맵 + 면접 질문`만으로 3만 원대 전략 리포트가 가능한가?
- 학생부 원본을 저장하지 않는 “분석 후 삭제” 구조가 초기 검증에 충분한가?
- 학부모 타겟 첫 채널은 맘카페/네이버/카카오 중 어디가 맞는가?
- AI기본법상 교육 분야 고영향 AI 오해를 피하려면 어떤 표현과 고지 문구가 필요한가?
